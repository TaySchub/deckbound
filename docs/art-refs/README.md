# art-refs — the developer's visual reference drop-box

Put images here that communicate taste better than words: mood references,
screenshots you liked, marked-up feedback (circle the thing, scribble the
note), and approved contact sheets.

**Why this folder exists:** re-explaining visual taste in chat every session is
the slowest part of the art loop. An image in this folder is permanent,
versioned direction that every future agent sees without being told.

## Conventions

- Name files `YYYY-MM-DD-topic.png` (e.g. `2026-07-10-steak-sear-reference.png`,
  `2026-07-10-feedback-slurper-cap.png`).
- When an art pass is approved, save its contact sheet as
  `approved-sheet-YYYY-MM-DD.png` — that becomes the visual baseline the next
  art PR's sheet is compared against (a poor man's visual regression test).
- Original or license-clean references only — same compliance rule as the art
  itself (`PROJECT.md` §6). References are inspiration for original work,
  never sources to copy.

## Agent duty

Before starting ANY art task: look at every image in this folder (newest
first), then read `docs/ART_STYLE.md`. If a reference and the style guide
conflict, the newer dated item wins — and flag the conflict in your PR.
